### 🚀 Today I Learned
#### OAuth
```
OAuth는 사용자가 자신의 비밀번호를 직접 제공하지 않고도 다른 웹사이트나 애플리케이션(이하 클라이언트)이 자신의 정보(리소스)에 접근할 수 있는 권한을 위임할 수 있도록 하는 개방형 표준(프로토콜)이다.
OAuth는 사용자 대신 클라이언트에게 제한된 접근 권한을 부여하여, 클라이언트가 사용자의 민감한 인증 정보를 알 필요 없이 특정 리소스(예: 이메일 주소, 프로필 정보, 캘린더 등)에 접근하게 해줍니다.

Resource Owner, Client, (Authorization Server, Resource Server) 로 구성됨
각각 사용자, 웹사이트, 클라이언트에게 접근 권한을 주는 곳, 사용자의 보호된 리소스 저장하는 곳

장점
보안성 강화: 클라이언트에게 사용자의 비밀번호를 직접 노출하지 않아 보안 위험을 낮춥니다.
제한적 접근: 발급된 토큰은 특정 리소스와 **제한된 권한(Scope)**만 가지므로, 클라이언트가 모든 정보에 무분별하게 접근하는 것을 방지합니다.
편의성: 사용자는 이미 계정이 있는 서비스(Google, Naver 등)를 통해 간편하게 다른 서비스에 로그인할 수 있어 사용자 경험이 향상됩니다.

```
