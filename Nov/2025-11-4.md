### 🚀 Today I Learned
#### 
```
💡 Hub(허브)

허브는 네트워크 장치 중 가장 단순하며
물리계층(Layer 1)에서 작동합니다.

여러 장치를 연결하고, 들어온 신호를 증폭하여 다른 모든 포트로 재전송(Broadcast)합니다.

데이터의 목적지 주소를 확인하지 않음

불필요한 트래픽 발생, 거의 사용되지 않음


💡 Switch(스위치)

스위치는 허브의 비효율성을 개선하기위해 등장
데이터 링크 계층(Layer 2)에서 작동

LAN내에서 장치 간의 데이터 패킷을 지능적으로 전달

데이터 프레임에 담긴 MAC 주소 학습 MAC 주소 테이블에 기록

데이터가 들어오면 이 테이블을 참조하여, 해당 MAC 주소를 가진 목적지 포트로만 데이터를 Unicast

불필요한 트래픽을 줄여 네트워크 성능을 크게 향상

L3 스위치는 IP 주소까지 인식하여 라우터의 일부 기능 수행하기도함


💡 Router(라우터)

라우터는 서로 다른 네트워크(LAN과 LAN)을 연결
데이터가 목적지까지 도달할 수 있는 최적의 경로 결정
네트워크 계층(Layer 3)에서 작동

데이터 패킷에 담긴 IP 주소(논리적 주소)를 읽음

라우팅 테이블을 참조하여 패킷을 보낼 다음 경로(Hop)을 결정하고 전송

서로 다른 IP 주소 대역을 연결하는 게이트웨이 

가정용 유무선 공유기가 라우터의 가장 흔한 형태


💡ARP(주소결정프로토콜)

네트워크에서 IP주소를 물리적 주소인 MAC주소와 연결 시키는데 사용되는 프로토콜

IP 주소를 알고 있을 때, 그 IP 주소에 해당하는 MAC주소를 알아내는 것

데이터를 보내려는 장치의 ARP request를 브로드캐스트
-> 해당 IP 주소를 가진 장치만이 요청을 받아 ARP Reply
-> 요청한 장치는 이 정보를 ARP 캐시 테이블에 두고 저장


💡 Symmetric ARP

목적지가 바뀜에 따라 포트도 바뀜

내부 호스트가 어떤 목적지와 통신하느냐에 따라 다른 외부 포트를 할당

외부공격자가 어떤 문을 알고 있어도, 그 문으로 통신을 요청했던 정해진 서버가 아니라면 절대로 내부로 접근할 수 없음(강력한 보안)

P2P 통신 매우 불리


💡 Cone ARP

외부 목적지가 달라도 포트는 같음

내부 호스트가 외부로 나갈 때, 목적지에 관계없이 포트 유지

한번 열린 문으로 엉뚱한 손님 침입 가능

P2P 통신 매우 유리

```
