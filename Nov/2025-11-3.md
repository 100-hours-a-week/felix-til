### **운영체제**

컴퓨터 하드웨어와 사용자 사이의 중재자 역할을 하는 중요한 시스템 소프트웨어(프로세스 관리, 메모리 관리, 파일/장치 관리)

### **리눅스**

운영체제의 한 종류, 커널에 기반을 둔 OS로 하드웨어 > 커널 > 셀(인터페이스) > 어플리케이션의 구조를 갖는다.

### User Mode & Kernel Mode

OS가 사용하는 CPU 실행 권한 레벨들로, 

User Mode의 주체는 애플리케이션이며 제한된 권한을 갖고 있어 하드웨어 자원이 필요한 경우 “시스템 호출”을 해야 한다.

Kernel Mode의 주체는 운영체제의 Kernel 이며 모든 시스템 자원에 직접 접근하고 제어하여 시스템의 안정성을 보장한다.

### 메모리 구조

코드 / 데이터 / 힙 / 스택으로 구성

코드(실행할 명령어 저장) / 데이터(전역변수, 정적변수 저장) / 힙(동적으로 할당되는 데이터 저장) / 스택(함수 호출 시 지역 변수, 매개변수, 복귀 주소 저장)

### 관리 기법

고정 분할: 메모리를 고정된 크기로 미리 나눔 → 내부 단편화 발생

가변 분할: 프로세스 크기에 맞춰 가변적으로 할당 → 외부 단편화 발생

페이징: 논리 주소(페이지)와 물리 주소(프레임)을 고정 크기로 나누어 비연속 할당 → 외부 단편화 제거

세그멘테이션: 프로그램을 논리적 단위로 나누어 가변 크기로 할당 → 외부 단편화 재발생

가상메모리: 프로세스에게 실제 물리적 메모리 크기보다 훨씬 큰 연속된 가상 주소 공간 제공

### 프로세스

실행 중인 프로그램

### 스레드

프로세스 내에서 실제로 작업을 실행하는 단위.

코드, 데이터, 힙 영역을 다른 스레드와 공유 / 스택과 레지스터는 개별 영역

### **동시성**

여러 작업을 동시에 처리하는 것처럼 보이게 만드는 것(시간을 나누어 번갈아 실행)

### 병렬성

여러 작업을 실제로 동시에 실행하는 것

### 스케줄러

시스템의 자원을 프로세스들에게 효율적이고 공평하게 할당하는 역할을하는 프로그램

### 공유자원

두 개 이상의 프로세스나 스레드가 동시에 접근하여 사용할 수 있는 데이터, 메모리 공간, 파일 또는 하드웨어 장치

### 임계영역

공유 자원에 접근하여 코드를 실행하는 프로그램의 일부분

### 동기화

멀티태스킹 환경에서 공유자원 접근 순서를 제어하여 데이터의 일관성을 유지하는 매커니즘

### 락

멀티태스킹 환경에서 공유자원에 대한 접근을 제어하여 동기화 문제를 해결하는 매커니즘

### 교착 상태

멀티태스킹 시스템에서 두 개 이상의 프로세스/스레드가 서로가 점유하고 있는 자원을 무한히 대기하는 상태

### 기아상태

특정 프로세스나 스레드가 시스템 자원을 계속 할당받지 못하고 영구적으로 대기하는 현상 → aging으로 해결

**세마포어**

공유자원 접근 제어와 프로세스와 스레드 간의 동기화를 위한 핵심적인 매커니즘
