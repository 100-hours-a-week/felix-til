### 🚀 Today I Learned
#### TCP/IP 모델
```
TCP (Transmission Control Protocol): 전송 계층의 프로토콜로, 신뢰성 있는 데이터 전송을 보장합니다. 데이터를 보내기 전에 연결을 설정하고(3-way-handshake), 데이터의 순서 및 오류를 제어합니다.
IP (Internet Protocol): 인터넷 계층의 프로토콜로, 패킷을 목적지까지 경로를 지정하여 전송하는 역할을 합니다.

응용 계층(Application Layer): "사용자와 가장 가까운 계층으로, 응용 프로그램이 네트워크 서비스를 이용할 수 있도록 합니다. (예: HTTP, FTP, DNS, SMTP)"
전송 계층(Transport Layer): "통신을 하는 프로세스(애플리케이션) 간의 논리적 연결 및 데이터 전송의 신뢰성을 담당합니다. (예: TCP, UDP)"
인터넷 계층(Internet Layer): "네트워크 간 패킷을 전송하며, 논리적 주소(IP 주소)를 기반으로 라우팅 경로를 결정합니다. (예: IP, ICMP, ARP)"
네트워크 접근 계층(Network Access Layer): "물리적인 매체를 통해 실제 데이터 전송을 담당하며, 물리적 주소(MAC 주소)를 사용합니다. (예: Ethernet, Wi-Fi)"
```
#### CIDR/서브넷
```
CIDR: IP 주소를 유연하게 할당/묶음으로써 IP 고갈 문제를 해결하고 라우팅 효율을 높인 기술/표기법입니다. (예: 192.168.1.0/24)
서브넷팅: 하나의 네트워크를 더 작은 여러 네트워크로 분할하여 효율적으로 관리하는 행위/과정입니다.
서브넷 마스크: 네트워크 ID와 호스트 ID를 구분하는 역할을 하는 값이며, CIDR 프리픽스(/n)를 통해 표현됩니다.

```
